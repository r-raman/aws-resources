MFA with CLI¶

The MFA is set under UAM/User/Security credentials tab Once the MFA is setup, we can get the arn of the MFA device

  1. To use MFA with the CLI, you must create a temporary session

  2. To create a temporary session use the STS GetSessionToken API call

aws sts get-session-token --serial-number arn-of-the-mfa-device --token-code code-from-token --duration-seconds 3600

Once you ger the MFA configured you can create an MFA profile

aws configure --profile mfa

You can input the values above in following prompt Then

cat ~/.aws/credentials

You should copy the token into the credentials

AWS Limits¶

API Rate limits

  1. DescribeInstances API for EC2 has a limit of 100 calls per second

  2. GetObject on S3 has a limit of 5500 GETs per second per prefix

  3. For Intermittent Errors – implement Exponential Backoff

  4. For Consistent Errors – request an API throttling limit increase

Service Quotas (formerly Service limits)

  1. Running on demand standard instances: 1152 vCPU

  2. For more vCPU, request a service limit increase by opening a ticket

  3. For service quota increase, use the service quota API

Exponential Backoffs

  1. They are used for avoiding intermittent errors

  2. When you get a throttling exceptions, use exponential backoff

  3. Must only retry 5xx errors

AWS CLI Credentials Provider Chain¶

https://docs.aws.amazon.com/cli/latest/userguide/cli-chap-configure.html#config-settings-and-precedence

The CLI will look for credentials in this order

  1. Command line options – –region, –output and –profile.

  2. Environment Variables – AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AND AWS_SESSION_TOKEN.

  3. CLI credentials file – aws configure ~/.aws/credentials.

  4. CLI configuration file – aws configure ~/.aws/config.

  5. Container credentials – for ECS tasks.

  6. Instance profile credentials – for EC2 instance profile.

For Java SDK (for example) will look for credentials in this order

  1. Java system properties – aws.accessKeyId and aws.secretKey

  2. Environment variables – AWS_ACCESS_KEY_ID AND AWS_SECRET_ACCESS_KEY

  3. The default system profile files – at ~/.aws/credentials

  4. Amazon ECS container credentials – for ECS containers

  5. Instance profile credentials – for EC2 instance profile

For with within AWS, use IAM roles

  1. EC2 instance roles for EC2 instances

  2. ECS roles for ECS tasks

  3. Lambda roles for Lambda functions

If working outside of AWS

  1. Use environment variables or named profiles

AWS Signing – CLI and the SDK

  1. When you call the AWS HTTP API, you sign the request so that AWS can identify you, using your AWS credentials (access key and secret key) Note: Some of the API request to S3 do NOT need to be signed

  2. If you use the SDK or the CLI, all the HTTP requests are signed for you

  3. If you are not using the SDK or the CLI to contact AWS, you should sign the HTTP request using Signature v4 (Sig v4)

Sig v4:

  1. Sign the request using the HTTP header

  2. Authrorization header

  3. The other option is to use the query string – S3 pre-signed URL

  • x-amz-algorithm

  • x-amz-credential

  • x-amz-date

  • x-amz-signature

When running the CLI on EC2 Instances, the CLI uses the ___meta___ service to get __temporary___ credentials thanks to the IAM Role that is attached.
